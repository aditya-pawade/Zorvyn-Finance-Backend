# Security Policy

## Supported Versions

The latest version on `main` is the supported version.

## Reporting a Vulnerability

If you find a security issue, please report it privately to the maintainer.

Include:

- Affected endpoint(s) or component
- Reproduction steps
- Impact summary
- Suggested mitigation (if available)

Please avoid opening public issues for active vulnerabilities until a fix is prepared.

## Security Notes

- Replace default JWT secrets before production use.
- Use strong database credentials and secure network access.
- Disable test-user seeding in production.
